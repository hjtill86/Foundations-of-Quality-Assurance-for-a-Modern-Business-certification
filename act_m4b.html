MODULE 4 | ACTIVITY B

Activity 4B — Document Control Case Study

← Back to Course Hub
🎯 Learning Objective: Analyze a document control failure, identify root causes, and recommend a corrective document management approach.

Scenario

Meridian Services is a professional services firm that recently underwent an external audit. The auditor found three employees using different versions of the same service delivery SOP — versions 1.2, 1.4, and 2.0. Version 2.0 was the current approved version. The superseded copies had not been removed from the shared drive. The audit resulted in a major nonconformance for failure to control documented information.

Question 1 — Root Cause Analysis

What were the root causes of this document control failure? Identify at least two distinct causes.

Question 2 — Corrective Action Plan

Design a document control procedure that would prevent this from recurring. Include at minimum: version numbering, distribution, and obsolescence handling.

Question 3 — ISO 9001 Alignment

Which ISO 9001:2015 clause does this nonconformance relate to, and what does that clause specifically require?

Model Answer

1Root Causes: (a) No formal document withdrawal procedure upon version update. (b) Open-access shared drive with no version control or archive segregation. (c) Staff not notified when new versions become effective.
2Corrective Procedure: All SOPs assigned a unique document number and version. Upon approval of a new version, the Document Controller immediately archives all previous versions to a read-only folder, updates the Master Document Register, and sends a controlled distribution notice to all relevant staff.
3ISO 9001 Clause 7.5.3 (Control of Documented Information) requires that documented information be protected from unintended alteration, that access is appropriate, and that obsolete versions are prevented from unintended use — exactly what failed here.
